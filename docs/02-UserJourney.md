# User Journey

## Goal

Enable teachers to create AI-powered interactive learning games with minimal effort and full control over the final experience.

---

## Primary Actor

### Teacher

Creates, reviews, edits, publishes, and evaluates learning games.

---

## Secondary Actor

### Student

Participates in and completes the learning games.

---

## Teacher Journey

### Step 1: Sign In

The teacher signs in to the platform.

---

### Step 2: Create a New Learning Game

The teacher starts creating a new learning game.

---

### Step 3: Fill the AI Game Creation Form

The teacher provides the minimum information required for AI to generate the learning game.

Required information:

* Grade Level
* Subject
* Topic
* Activity Types
* Indoor or Outdoor
* Number of Students

The teacher selects one or more activity types:

* Quiz
* QR Code
* GPS

The teacher also decides how many tasks of each type should be generated.

Example:

* 3 Quiz Tasks
* 2 QR Code Tasks
* 2 GPS Tasks

---

### Step 4: AI Generates the Learning Game

AI generates the first draft of the learning game.

The AI respects the structure defined by the teacher and creates the game content accordingly.

---

### Step 5: Review

The teacher reviews the generated learning game.

---

### Step 6: Edit

The teacher can modify, remove, or add any generated content.

The teacher always has full control over the final version of the game.

---

### Step 7: Publish

The teacher publishes the learning game.

Publishing prepares the game for student participation.

---

### Step 8: Share

The teacher activates the learning game and shares the Game Code with students.

---

### Step 9: Play

Students participate in the learning game using their mobile devices.
Students join the learning game by entering the Game Code and their names.

Students complete the selected activities:

* Quiz Tasks
* QR Code Tasks
* GPS Tasks

Students earn points by completing activities and answering questions correctly.

---

### Step 10: Review Results

The teacher reviews:

* Student Participation
* Scores
* Progress
* Learning Results

The teacher uses these insights to evaluate the learning activity.

---

## User Journey Principles

* Teachers provide only the minimum required information.
* Teachers define the structure of the learning game.
* AI generates the game content.
* AI creates the first draft.
* Teachers stay in control.
* Teachers can modify any AI-generated content.
* Students enjoy an interactive learning experience.

---

## Expected Outcome

A teacher should be able to create, customize, publish, and evaluate an AI-powered learning game through a simple and guided workflow.
